Card Transaction Processor — API (local)

This page is served in development for quick reference. All JSON APIs (except POST /api/v1/auth/login and this help page) require Authorization: Bearer <accessToken>.

Base URL

Default: http://localhost:9091 (override with SERVER_PORT / server.port).

Authentication

Login with the seeded user (or your own DB user), then use the returned token.

curl -s -X POST http://localhost:9091/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"YOUR_PASSWORD"}'

Passwords are the BCrypt hashes in server/scripts/sql/09_seed_admin_user.sql; use the plaintext you encoded in Java when logging in (and in Postman seedPassword). Response includes accessToken and expiresIn (seconds).

On subsequent calls:

curl -s http://localhost:9091/api/v1/auth/me \
  -H "Authorization: Bearer <accessToken>"

Errors

Failed requests return JSON: {"code":"…","message":"…","traceId":null,"details":…} (see ApiErrorDto). 401 = missing/invalid token; 403 = missing permission/role; 400 = validation or domain rule.

Permission column is the hasAuthority('…') name required. POST /login is public.

Environment

Typical: DB_URL, DB_USERNAME, DB_PASSWORD, JWT_SECRET (≥ 32 bytes). See server/processor-api/src/main/resources/application.yml.

CORS: local Vite (5173) and common dev ports are allowed. API calls from a browser on another origin must use these origins or add them in SecurityConfig.
